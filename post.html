What I Learned Building My First Web App

January 15, 2025 · 6 min read · Your Name

Six months ago I decided to build my first "real" web app — not a tutorial project, but something I actually wanted to exist. This is the honest account of what happened, what I broke, and what I'd do differently.

The Idea

My idea was simple: a task manager that grouped tasks by project. Basic stuff. I figured it would take two weeks. It took two months.

That gap between "how long I thought it would take" and "how long it actually took" is the most universal experience in software development, and nobody tells you this before you start.

Mistake #1: Skipping the Plan

I opened VS Code and started typing HTML immediately. No wireframe, no data model, nothing. By week two I had spaghetti code and was rewriting half of it.

Spending 30 minutes planning can save you 3 hours of refactoring. I wish someone had told me this on day one.

Now I sketch on paper first. Even a rough box-and-arrow diagram of how the data flows makes a huge difference.

Mistake #2: Not Using Git From Day One

I started using Git only after two weeks of coding. That means my early history is gone. Now I git commit after every meaningful change, even tiny ones. 

# The commands I use every single day now
git add .
git commit -m "feat: add task filtering by project"
git push origin main

What I'd Tell Myself

  • Start smaller than you think. Cut the scope in half, then cut it again.
  • Read error messages carefully. They usually tell you exactly what's wrong.
  • Git early, Git often. You'll thank yourself later.
  • Show your work to others. Even if it's embarrassing. Feedback is gold.

The project isn't perfect. But it exists, it works, and I learned more from building it than from any course I've taken. That's the whole point.
